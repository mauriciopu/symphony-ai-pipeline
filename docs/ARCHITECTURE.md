# Architecture

## Overview

Symphony is a **code-enforced multi-agent orchestration framework** for Claude Code. It coordinates 7 specialized agents through a persistent state machine, ensuring every task passes mandatory quality gates.

## Design Principles

### 1. Code Enforces, Not LLM Discipline
LLMs are unreliable at following multi-step procedures. Symphony enforces phase ordering through bash hooks that read `pipeline-state.json` and reject invalid operations. The LLM literally cannot skip phases — the hook blocks the command.

### 2. Persistent State Survives Crashes
All state lives in JSON files, not LLM memory. If a session crashes, the next session reads `pipeline-state.json` and resumes from exactly where it left off.

### 3. Cold-Start Agents
Each subagent (coder, tester, reviewer) gets a fresh context per task. No state carries between tasks. The task description in Linear contains everything the agent needs. This makes agents disposable and debuggable.

### 4. Immutable Audit Trail
Every state machine transition is logged to `pipeline-audit.jsonl`. This provides post-mortem analysis capability and holds the coordinator accountable.

## Components

### State Machine (`pipeline-advance.js`)
The backbone. 15 actions, validated transitions, persistent state.

```
idle → phase0_mcp → phase1_discovery → phase1.5_coverage →
phase2_task_coder → phase2_task_tester → phase2_task_reviewer →
phase2_task_commit → [loop or] →
phase3_integration → phase4_auditor → phase5_pr → phase6_reconcile → idle
```

Phase 2 is a loop: for each Story, for each Task, run coder→tester→reviewer→commit.

### Hook Composition
Six PreToolUse hooks run on every Bash command. Order matters:
1. `prevent-dangerous-ops.sh` — safety net (blocks rm -rf, force push, etc.)
2. `architecture-boundary.sh` — DDD enforcement
3. `auth-coverage.sh` — OWASP auth enforcement
4. `test-quality.sh` — no unjustified test skips
5. `feedback-enforcer.sh` — learned rules from past mistakes
6. `pipeline-gate.sh` — state machine phase enforcement

Two PostToolUse hooks run after every tool call:
1. `post-tool-telemetry.sh` — dashboard heartbeat
2. `deferred-work-check.sh` — dependency unblocking

### Agent Hierarchy
```
Coordinator (opus, level 0)
  ├── Coder (sonnet, level 1) — writes code
  ├── Tester (haiku, level 1) — validates gates
  ├── Reviewer (sonnet, level 1) — quality/security gate
  ├── Auditor (opus, level 1) — compliance audit
  ├── DevOps (haiku, level 1) — PR management
  └── PR Validator (opus, level 1) — auto-repair loop
```

Model selection is cost-aware:
- **Opus**: coordinator + auditor (complex orchestration, compliance)
- **Sonnet**: coder + reviewer (code generation, analysis)
- **Haiku**: tester + devops (command execution, simple tasks)

### Quality Gates (11 total)
| Gate | Type | What it checks |
|------|------|---------------|
| Build | Blocking | Zero compilation errors |
| TypeScript | Blocking | Zero type errors |
| Lint | Blocking | Zero style violations |
| Tests | Blocking | All pass, 80%+ coverage |
| Architecture | Blocking | DDD boundaries respected |
| Auth | Blocking | No accidentally public endpoints |
| Test Quality | Blocking | No unjustified skips |
| Any Types | Blocking | No `any` in production code |
| MCP Health | Informative | All MCP servers reachable |
| Pipeline State | Informative | Idle or active with context |
| Tracker Hygiene | Informative | No stale in-progress issues |

### Dashboard
Real-time HTML monitor (auto-refresh 5s):
- Phase flow visualization
- Quality gate status
- Task/story progress
- Audit log
- Recent tool activity

Generated by `generate-dashboard.js` from `pipeline-status.json`. Updated by `post-tool-telemetry.sh` on every tool call.

## Data Flow

```
User → /start-unit → Coordinator
  │
  ├─ Reads: pipeline-state.json (resume or fresh start)
  ├─ Reads: Linear issues (hierarchy discovery)
  ├─ Reads: Code generation plan
  │
  ├─ For each Task:
  │   ├─ Writes: Linear (state: "In Progress")
  │   ├─ Spawns: Coder agent → writes code + tests
  │   ├─ Spawns: Tester agent → validates 5 gates
  │   ├─ Spawns: Reviewer agent → quality/security review
  │   ├─ Writes: git commit
  │   ├─ Writes: pipeline-state.json (task-done)
  │   ├─ Writes: pipeline-audit.jsonl (append)
  │   └─ Writes: Linear (state: "Done" + workpad)
  │
  ├─ Spawns: Auditor agent → 38-point compliance check
  ├─ Writes: git push + PR via GitHub MCP
  ├─ Writes: Linear reconciliation (all 3 levels)
  └─ Writes: pipeline-state.json (complete → idle)
```
